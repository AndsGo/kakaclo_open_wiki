# 小技巧

**判断AI是否理解需求**：\
你看对于这个功能你还有没有什么不明白的，有的话你提出来，如果没有的话，你先给我列一下，实现这个功能，前后端的一个任务列表，没问题的话再介入开发。

**结构化提取图片中的UI/UX:**

创建一个 JSON 格式的设计系统配置文件。该配置文件应从提供的屏幕截图中提取相关的视觉设计信息。JSON 输出必须具体包含：\
总体设计风格（例如，调色板、排版、间距、视觉层次结构）。

结构元素和布局原则。

任何其他对 AI 一致复制这些设计系统至关重要的属性。

至关重要的是，不要包含图像中存在的具体内容或数据，而仅关注设计特征。

**AI工作方法：把AI当人，并且给予它明确的工作流：**

* 第一步：理解AI产品的规则，**要充分再充分**。Claude官网有大量使用介绍，prompt模板。同时，官网介绍里有7\*24小时的规则，所以他认为7×24小时是非常合规的，他最多只是熟读官网利用了机会。
* 第二步：把AI当做真实的“程序员”提需求，不是给AI提示词，他的方法是用飞书写一个1000-2000字需求文档，甚至画一些图形，投喂AI；然让AI**复述**文档里的需求，通过和AI交互中让需求更准确与全面，最多一次与AI交互65轮需求。保证需求被理解了，这样能最小化减少AI理解偏差。
* 第三步：**设计流程让AI持续工作**。Claude code本身是一个agent，假设让“设计画图APP的一个齐白石风格”，会让AI学习设计师的工作流：
  * ①在网站学习齐白石的介绍，学习齐白石画作风格；
  * ②开始让AI画图，将画图工具 API给到 AI，每个风格先画9张图，最后设计成icon图；
  * ③再让AI自己选每个风格最佳的一个，过程中AI如果拿不准就招呼人去解决。
