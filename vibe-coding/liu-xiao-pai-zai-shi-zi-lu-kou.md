# 刘小排在“十字路口”

原文:[https://mp.weixin.qq.com/s/f3mY6Xcne-L\_iMpf4FGc8w](https://mp.weixin.qq.com/s/f3mY6Xcne-L_iMpf4FGc8w)\
**1、哪些工作可以被AI 替代？**\


一切可以SOP的事情，AI编程都可以做，所以大多非物理世界（互联网）的事大概率都会被AI重新洗牌；提供情趣价值或者奖励信号比较稀疏的工作很难被替代。

**2、与AI交互更好的最佳方法：**

转变理念：不要把AI当工具，而是理解通晓机器的语言，才能更好的把机器为我作用。人与人之间的差异，都需要花费时间钻研，机器的运作方式更就应该钻研。\


从今天起，接受人机共生吧，用刘小排的话是 “it is not for you，it is you 它就是你的一部分”。

**AI工作方法：把AI当人，并且给予它明确的工作流：**

* 第一步：理解AI产品的规则，要充分再充分。Claude官网有大量使用介绍，prompt模板。同时，官网介绍里有7\*24小时的规则，所以他认为7×24小时是非常合规的，他最多只是熟读官网利用了机会。
* 第三步：把AI当做真实的“程序员”提需求，不是给AI提示词，他的方法是用飞书写一个1000-2000字需求文档，甚至画一些图形，投喂AI；然让AI复述文档里的需求，通过和AI交互中让需求更准确与全面，最多一次与AI交互65轮需求。保证需求被理解了，这样能最小化减少AI理解偏差。
* 第二步：设计流程让AI持续工作。Claude code本身是一个agent，假设让“设计画图APP的一个齐白石风格”，会让AI学习设计师的工作流：
  * ①在网站学习齐白石的介绍，学习齐白石画作风格；
  * ②开始让AI画图，将画图工具 API给到 AI，每个风格先画9张图，最后设计成icon图；
  *   ③再让AI自己选每个风格最佳的一个，过程中AI如果拿不准就招呼人去解决。

      \


最终可以设计900个方案，过程中自己可能都在睡觉。

**3、如何将AI压榨到最大化：**

* 他举了一个例子，将一个产品语言翻译成32个文字。如果从节省token的方法，应该先让claude写个翻译脚本，然后再进行翻译，但为了省自己时间，还是会被任务直接给到claude，让它自己一步步运营。
*   奉行实用主义，而非夸夸其谈：有些人在评点别人为人的时候，他已经变现了好一阵。

    \


**4、不同AI有不同的长板，要组合式使用：谷歌Gemini pro适合做架构师，gpt5适合解难题，日常coding 交给claude code。**

**5、产品心得**：

* AI套壳网站是公司目前是最挣钱的业务，尤其实一开始能定位用户需求的公司，就不要营销费用，这是和大厂的本质区别。
*   产品的最关键是「简单」，产品有一个按钮的时候，就不要有两个按钮。傅盛老师产品三段论：预测、单点击穿、all in，不断找用户痛点，做出mvp去验证。 可行放大，不可行放弃。

    \


**6、专注很重要**：刘小排虽然过往股票战绩不错，自己在做公司之后，就将股票账号清空了，为了节省精力用在创业上。

**7、走出来混是最重要的**：机会的源头是走出来，参加了傅盛的创业营被收购，同期还有Musically创始团队（TikTok 前身）。
